# 노션 기반 견적서 공유 서비스 MVP PRD

## 핵심 정보

**목적**: 노션 데이터베이스를 CMS로 사용하여 관리자가 작성한 견적서를 클라이언트에게 웹 링크로 공유하고 PDF로 다운받을 수 있게 한다
**사용자**: 관리자(견적서 작성자), 클라이언트(견적서 수신자)
**목표**: 최소 기능으로 빠른 출시 후 사용자 피드백 기반 개선

---

## 🗂️ 노션 데이터 구조

노션에서 아래 두 가지를 관리한다. 별도 데이터베이스(Supabase 등)는 사용하지 않는다.

### 노션 데이터베이스 (견적서 목록)

관리자가 노션에서 직접 관리하는 데이터베이스 페이지:

| 필드 | 설명 | 노션 속성 타입 |
|------|------|--------------|
| 제목 | 견적서 이름 | Title |
| slug | 공유 URL용 고유 슬러그 | Text |
| page_id | 견적서 내용 노션 페이지 ID | Text |

### 노션 페이지 (견적서 내용)

실제 견적서 내용이 담긴 노션 페이지. 관리자가 자유롭게 작성.

---

## 🚶 사용자 여정

### 관리자 여정

```
1. 노션에서 견적서 내용 페이지 작성

2. 노션 데이터베이스에 항목 추가
   - 제목, slug(예: client-abc-2026), page_id 입력

3. 클라이언트에게 공유 URL 전달
   예: https://invoice.example.com/invoice/client-abc-2026
```

### 클라이언트 여정

```
1. 이메일/메시지로 받은 공유 링크 클릭

2. 견적서 공개 조회 페이지
   ↓ 웹이 노션 API로 slug → page_id 조회 후 페이지 블록 렌더링

   [유효한 링크] → 견적서 내용 표시
   [존재하지 않는 slug] → 오류 안내 페이지

3. "PDF 다운로드" 버튼 클릭
   ↓ PDF 생성 및 다운로드

4. 완료
```

## ⚡ 기능 명세

### 1. MVP 핵심 기능

| ID | 기능명 | 설명 | MVP 필수 이유 | 관련 페이지 |
|----|--------|------|-------------|------------|
| **F001** | slug로 견적서 조회 | 노션 데이터베이스에서 slug로 page_id 조회 | 공유 URL → 콘텐츠 연결의 핵심 | 견적서 공개 조회 페이지 |
| **F002** | 노션 페이지 실시간 렌더링 | Notion API로 페이지 블록을 읽어 웹에서 렌더링 | 서비스의 핵심 가치 | 견적서 공개 조회 페이지 |
| **F003** | PDF 다운로드 | 렌더링된 견적서를 PDF로 변환하여 다운로드 | 클라이언트의 핵심 니즈 | 견적서 공개 조회 페이지 |

### 2. MVP 이후 기능 (제외)

- 회원가입 / 로그인 / 인증 시스템
- 별도 데이터베이스 (Supabase 등) 사용
- 견적서 만료일 설정
- 견적서 열람 횟수/통계 확인
- 비밀번호로 견적서 보호
- 견적서 커스텀 브랜딩 (로고, 색상)
- 이메일 직접 발송 기능
- 알림 기능

## 📄 페이지별 상세 기능

### 견적서 공개 조회 페이지

> **구현 기능:** `F001`, `F002`, `F003` | **인증:** 불필요 (공개 링크)

| 항목 | 내용 |
|------|------|
| **역할** | 클라이언트가 공유 URL로 접속하여 견적서를 확인하고 PDF를 다운받는 핵심 공개 페이지 |
| **진입 경로** | 관리자에게 전달받은 공유 URL 직접 접속 (`/invoice/[slug]`) |
| **사용자 행동** | 견적서 내용을 웹에서 확인하고 PDF 다운로드 버튼 클릭 |
| **주요 기능** | slug로 노션 DB 조회 → page_id 확보 (F001), Notion API로 페이지 블록 실시간 렌더링 (F002), **"PDF 다운로드"** 버튼 클릭 시 PDF 변환 및 저장 (F003) |
| **다음 이동** | 유효한 slug → 견적서 렌더링 표시, 존재하지 않는 slug → 오류 안내 페이지 |

---

### 오류 안내 페이지

> **구현 기능:** 없음 | **인증:** 불필요

| 항목 | 내용 |
|------|------|
| **역할** | 존재하지 않는 slug 접근 시 안내를 제공하는 오류 페이지 |
| **진입 경로** | 노션 DB에 없는 slug로 접속 시 자동 이동 |
| **사용자 행동** | 오류 내용 확인 후 이탈 |
| **주요 기능** | "이 견적서는 존재하지 않거나 삭제되었습니다" 안내 메시지 |

## 🛠️ 기술 스택

### 🎨 프론트엔드 프레임워크

- **Next.js 15** (App Router, Turbopack) - React 풀스택 프레임워크
- **TypeScript 5** - 타입 안전성 보장
- **React 19** - UI 라이브러리

### 🎨 스타일링 & UI

- **TailwindCSS v4** - 유틸리티 CSS 프레임워크
- **shadcn/ui** (radix-nova 스타일) - 고품질 React 컴포넌트 라이브러리
- **Lucide React** - 아이콘 라이브러리

### 🔗 외부 연동

- **Notion API** (@notionhq/client) - 환경변수 API Key로 노션 DB 조회 및 페이지 블록 읽기
- **notion-to-md** 또는 **react-notion-x** - 노션 블록을 웹으로 렌더링

### 📄 PDF 생성

- **html2canvas + jsPDF** - 렌더링된 견적서를 PDF로 변환

### 🚀 배포 & 호스팅

- **Vercel** - Next.js 15 최적화 배포 플랫폼

### 📦 패키지 관리

- **npm** - 의존성 관리
