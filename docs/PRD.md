# 노션 기반 견적서 공유 서비스 MVP PRD

## 핵심 정보

**목적**: 노션 데이터베이스를 CMS로 사용하여 관리자가 작성한 견적서를 클라이언트에게 웹 링크로 공유하고 PDF로 다운받을 수 있게 한다
**사용자**: 관리자(견적서 작성자), 클라이언트(견적서 수신자)
**목표**: 최소 기능으로 빠른 출시 후 사용자 피드백 기반 개선

---

## 🗂️ 노션 데이터 구조

노션에서 아래 두 개의 데이터베이스를 관리한다. 별도 데이터베이스(Supabase 등)는 사용하지 않는다.

### 1. 견적서 DB (Invoice Database)

관리자가 견적서 단위로 관리하는 데이터베이스:

| 필드 | 설명 | 노션 속성 타입 |
|------|------|--------------|
| `Name` | 견적서 이름 | Title |
| `slug` | 공유 URL용 고유 슬러그 (예: `client-abc-2026`) | Text |
| `client` | 클라이언트 회사/이름 | Text |
| `date` | 견적서 발행일 | Date |
| `total` | 합계 금액 (수동 입력 또는 Rollup 자동계산) | Number |
| `items` | 항목 DB와 연결 (Relation) | Relation → Items DB |

### 2. 항목 DB (Items Database)

견적서에 포함되는 개별 항목을 관리하는 데이터베이스:

| 필드 | 설명 | 노션 속성 타입 |
|------|------|--------------|
| `Name` | 항목명 (서비스/상품 이름) | Title |
| `quantity` | 수량 | Number |
| `unit_price` | 단가 (원) | Number |
| `amount` | 금액 (`quantity × unit_price` 자동계산) | Formula |
| `invoice` | 견적서 DB와 연결 (역방향 Relation) | Relation → Invoice DB |

### 데이터 흐름

```
[견적서 DB] ──Relation──▶ [항목 DB]
  slug: client-abc-2026      항목1: 기획 10h × 100,000
  client: ABC 클라이언트      항목2: 디자인 5h × 120,000
  date: 2026-06-26           항목3: 개발 20h × 150,000
  total: 4,100,000
```

**API 조회 흐름**: `slug`로 견적서 DB 조회 → Relation의 item page_id 배열 확보 → 각 항목 DB 페이지 조회 → 견적서 웹 렌더링

## 🚶 사용자 여정

### 관리자 여정

```
1. 노션 항목 DB(Items)에 견적 항목 추가
   - 항목명, 수량, 단가 입력 → amount 자동계산

2. 노션 견적서 DB(Invoice)에 견적서 추가
   - Name(제목), slug, client, date, total 입력
   - items Relation으로 위에서 만든 항목들 연결

3. 클라이언트에게 공유 URL 전달
   예: https://invoice.example.com/invoice/client-abc-2026
```

### 클라이언트 여정

```
1. 이메일/메시지로 받은 공유 링크 클릭

2. 견적서 공개 조회 페이지
   ↓ 웹이 노션 API로 slug → 견적서 정보 + 항목 목록 조회 후 렌더링

   [유효한 링크] → 견적서 내용 표시 (항목 테이블 + 합계)
   [존재하지 않는 slug] → 오류 안내 페이지

3. "PDF 다운로드" 버튼 클릭
   ↓ PDF 생성 및 다운로드

4. 완료
```

## ⚡ 기능 명세

### 1. MVP 핵심 기능

| ID | 기능명 | 설명 | MVP 필수 이유 | 관련 페이지 |
|----|--------|------|-------------|------------|
| **F001** | slug로 견적서 조회 | 견적서 DB에서 slug로 견적서 정보 및 연결된 항목 ID 목록 조회 | 공유 URL → 콘텐츠 연결의 핵심 | 견적서 공개 조회 페이지 |
| **F002** | 견적서 + 항목 렌더링 | 견적서 정보(client, date, total)와 Relation으로 연결된 항목 목록(name, quantity, unit_price, amount)을 웹에서 렌더링 | 서비스의 핵심 가치 | 견적서 공개 조회 페이지 |
| **F003** | PDF 다운로드 | 렌더링된 견적서를 PDF로 변환하여 다운로드 | 클라이언트의 핵심 니즈 | 견적서 공개 조회 페이지 |

### 2. MVP 이후 기능 (제외)

- 회원가입 / 로그인 / 인증 시스템
- 별도 데이터베이스 (Supabase 등) 사용
- 견적서 만료일 설정
- 견적서 열람 횟수/통계 확인
- 비밀번호로 견적서 보호
- 견적서 커스텀 브랜딩 (로고, 색상)
- 이메일 직접 발송 기능
- 알림 기능

## 📄 페이지별 상세 기능

### 견적서 공개 조회 페이지

> **구현 기능:** `F001`, `F002`, `F003` | **인증:** 불필요 (공개 링크)

| 항목 | 내용 |
|------|------|
| **역할** | 클라이언트가 공유 URL로 접속하여 견적서를 확인하고 PDF를 다운받는 핵심 공개 페이지 |
| **진입 경로** | 관리자에게 전달받은 공유 URL 직접 접속 (`/invoice/[slug]`) |
| **사용자 행동** | 견적서 내용을 웹에서 확인하고 PDF 다운로드 버튼 클릭 |
| **주요 기능** | slug로 견적서 DB 조회 → 견적서 정보 + items Relation page_id 목록 확보 (F001), 항목 DB 조회 후 견적서 UI 렌더링—항목 테이블 + 합계 (F002), **"PDF 다운로드"** 버튼 클릭 시 PDF 변환 및 저장 (F003) |
| **다음 이동** | 유효한 slug → 견적서 렌더링 표시, 존재하지 않는 slug → 오류 안내 페이지 |

---

### 오류 안내 페이지

> **구현 기능:** 없음 | **인증:** 불필요

| 항목 | 내용 |
|------|------|
| **역할** | 존재하지 않는 slug 접근 시 안내를 제공하는 오류 페이지 |
| **진입 경로** | 노션 DB에 없는 slug로 접속 시 자동 이동 |
| **사용자 행동** | 오류 내용 확인 후 이탈 |
| **주요 기능** | "이 견적서는 존재하지 않거나 삭제되었습니다" 안내 메시지 |
